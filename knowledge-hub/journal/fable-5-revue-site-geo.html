Fable 5 passe le site en revue : 4 bugs, du RSS et une veille enfin visible des LLMs

Claude · HL · 10 juin 2026
meta seo-gtm agent en-prod Perso

Contexte

Anthropic a sorti Claude Fable 5 le 9 juin — premier modèle d'une nouvelle classe « Mythos », au-dessus d'Opus (j'ai écrit un guide sur toute l'histoire Fable/Mythos/Glasswing). Inclus dans l'abonnement jusqu'au 22 juin : autant le tester sur un cas réel. Le test choisi : une revue complète de ce site — technique, SEO, GEO, automatisations — avec mandat de pousser les corrections en prod.

Ce qui a été fait

L'audit a d'abord rendu un verdict rassurant (robots.txt ouvert aux crawlers IA, llms.txt, JSON-LD, miroir EN complet : le socle GEO tient), puis a trouvé quatre vrais bugs que personne n'avait vus :

  • le compteur dynamique de la home comptait la page d'index du journal comme une session (45 affiché au lieu de 44) ;
  • le script d'orthographe US avait cassé un slug dans une URL de l'index de recherche EN (analyse → analyze = lien mort) ;
  • une entrée de journal manquait entièrement dans l'index de recherche EN ;
  • la home EN filtrait kind === 'Projet' sur l'index… anglais, où le kind s'appelle Project.

Côté GEO, la trouvaille la plus utile : la veille IA était invisible pour les LLMs. La page rend le feed en JavaScript, or GPTBot, ClaudeBot et PerplexityBot n'exécutent pas le JS — mon contenu le plus frais n'existait pas pour eux. Fix : un script qui injecte les 10 derniers jours en HTML statique dans la page (le JS l'écrase chez les visiteurs, zéro changement visuel). Ajoutés aussi : deux flux RSS (journal + veille), des llms.txt datés, et un check de cohérence hebdomadaire (GitHub Actions) qui vérifie fichiers vs index vs compteurs — le genre de garde-fou qui aurait attrapé les quatre bugs ci-dessus.

La session a continué sur la lancée : llms-full.txt est désormais auto-généré depuis les pages du site (méthode, guides, lexique, les 45 entrées de journal — absentes de l'ancienne version manuelle) par un workflow qui tourne à chaque push ; la home affiche la veille du jour en HTML statique, régénérée automatiquement quand le bot pousse le feed ; et l'annonce publique a été préparée dans la même session — premier post LinkedIn + un carrousel de 8 slides aux couleurs du design system du site, généré en HTML puis exporté en PDF.

Résultat & suite

Tout est parti en prod dans la session, avec la procédure documentée dans le CLAUDE.md du repo (Compound Engineering : chaque correctif devient une règle ou un script). Première impression de Fable 5 sur ce genre de mission : l'audit va chercher des causes (pas juste des symptômes — le slug cassé a été remonté jusqu'au script fautif), et la session enchaîne audit → fix → prod sans se perdre.

La suite : retour sur des modèles moins chers pour le quotidien — Fable se garde pour les problèmes durs, c'est exactement la conclusion du guide. Et le prochain chantier est déjà ouvert : les loops, ces tâches récurrentes d'agent que je déploie en ce moment chez Cobra.
