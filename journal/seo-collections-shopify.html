Cobra · SEO bulk — correction de masse

Correctif SEO collections Shopify (audit Guillaume)

Claude · Cobra · mai 2026 · Cobra
seo-gtmshopifyen-prod

Contexte

~175 pages collections cobra.fr avaient été mass-optimisées deux semaines avant. Guillaume (SEO) a audité et identifié 11 problèmes systémiques : titles trop uniformes, marques en H1, arguments répétés, suppression erronée du signal local « Paris 11 », suffix | Cobra.fr partout.

Ce qui a été construit

  • Phase 1 — Top 8 collections : H1, meta title et meta description réécrits selon la typologie Guillaume (Type A/B) — 0 erreur, une mutation batched de 8 alias
  • Phase 2 — Restauration « Paris 11 » : descriptionHtml corrigé via Python + mutation GraphQL avec variables
  • Phase 3 — Suppression bulk | Cobra.fr : pagination complète (8 pages × 50), 182 collections, 26 batches de 7 mutations aliasées — 182/182, 0 erreur

Ce qui était difficile

  • bodyHtml n'existe pas sur les collections — c'est descriptionHtml. Et title contrôle H1 + fil d'Ariane + menu + og:title : distinguer title (navigation) et seo.title (balise <title>)
  • Session démarrée après compaction (résumé de 8 000 tokens) : tout l'état reconstruit depuis le résumé

Stack

Shopify Admin GraphQL (mutations batched avec alias c1:…c7:), Python (re, json), ShopifyQL.

Ce que ça illustre

Correction SEO post-optimisation de masse : ne pas tout uniformiser, adapter chaque balise à l'intention, gérer les effets de bord Shopify. Reprise d'un chantier après compaction, 26 batches sans erreur. Pattern « valider la structure une fois, exécuter N fois ».
