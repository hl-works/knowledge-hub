Structuration projet Claude « Odoo » : docs de référence écosystème

Claude · Cobra · 11 mai 2026 · Cobra
metaen-prod

Contexte

Le projet Claude dédié à Odoo était vide (juste logos + charte). Manquait un document de référence consolidant toutes les connexions techniques validées pour que chaque conversation démarre avec le bon contexte sans tout ré-expliquer.

Ce qui a été construit

  • cobra_connexions_ecosysteme.md — 165 lignes : instance Odoo SaaS, GitHub, Shopify, Irokoo, outils locaux, snapshot des 9 modules custom, numérotation CDC, stratégie d'autonomie
  • Prompt prêt à l'emploi pour lancer la conversation sur cobra_field_security (CDC-18)
  • Recommandation validée : tout garder dans le projet Odoo existant

Ce qui était difficile

  • La distinction Odoo SaaS vs Odoo.sh méritait d'être explicitée noir sur blanc
  • La stratégie « passer un max en prod sans Irokoo » change le réflexe par défaut (natif Odoo d'abord, CDC en dernier recours)

Stack

Markdown, projet Claude (knowledge base), mémoire conversation.

Ce que ça illustre

Claude comme architecte de sa propre base de connaissance projet. Pattern « session de structuration » avant de se lancer dans le dev : souvent rentabilisé dès la session suivante.
