Perso · Méta — structuration du système Claude

Setup Claude : documentation, onboarding et Knowledge Hub

Claude · HL · 30 mai 2026 · Perso
metaen-cours

Contexte

Besoin de documenter la logique du setup Claude (fichiers .md, projets, deux comptes) pour deux raisons : transmettre le système à un ami qui démarre (Romain / Ballard), et se donner un journal centralisé pour capitaliser session après session.

Ce qui a été construit

  • Explication complète du système de fichiers : CLAUDE.md, CLAUDE-strat.md, CLAUDE-dev.md, histoire_cobra.md, claude-cobra-odoo.md
  • Distinction Claude Chat vs Claude Code et quand utiliser l'un ou l'autre
  • Logique d'appel conditionnel des extensions
  • Email complet pour Romain (Ballard) : explication du système + prompt clé en main pour son CLAUDE.md et son fichier ADN entreprise
  • 3 prompts réutilisables (résumé de session, onboarding d'un nouveau Claude, extraction de contexte)
  • Brief complet pour Claude Code pour construire ce repo

Ce qui était difficile

  • Le « sync automatique » Claude ↔ GitHub n'est pas faisable nativement — la solution semi-manuelle (résumé → copier-coller → git push) suffit largement
  • La distinction public/privé sur GitHub Pages n'est pas une vraie auth — du « non référencé » assumé, suffisant pour des .md non sensibles

Stack

Markdown, GitHub Pages, HTML/CSS statique — session de structuration et documentation.

Ce que ça illustre

Claude utilisé en méta-outil — pas pour produire du code, mais pour documenter une pratique, la transmettre, et poser les bases d'un système qui s'améliore dans le temps. Le Compound Engineering appliqué à lui-même.
