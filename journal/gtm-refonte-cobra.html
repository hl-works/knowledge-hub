Cobra · Audit + migration GTM en temps réel

Refonte GTM cobra.fr : purge 71 → 20 balises + Consent Mode v2

Claude · Cobra · mai 2026 · Cobra
seo-gtmen-prod

Contexte

GTM cobra.fr : 71 balises accumulées sur des années, CMP Axeptio obsolète, dizaines de variables orphelines (Criteo, RTBHouse, Cobrason, Salecycle…), Consent Mode v1 bricolé via cookies. Objectif : architecture propre avec Consent Mode v2 natif Shopify et Shopify Pixels.

Ce qui a été construit

  • Migration Consent Mode v2 via visitorConsentCollected Shopify natif (v178)
  • Purge 71 → 20 balises actives (v179–v181)
  • Suppression ~70 variables orphelines
  • Réactivation Google Ads Conversion (v182), recréation Idealo (v183)
  • Migration Microsoft Ads UET + Conversions vers Consent Mode v2 (v184)

Ce qui était difficile

  • Les variables Cobrason bloquaient la suppression (référencées par des variables intermédiaires orphelines) — chaîne de dépendances à remonter manuellement
  • Google Tag Gateway semblait faisable mais c'est le Cloudflare de Shopify sans accès admin — voie abandonnée

Stack

Google Tag Manager, Shopify Pixels natifs, Consent Mode v2, DataLayer, JavaScript personnalisé.

Ce que ça illustre

GTM assisté en temps réel : Claude guide étape par étape, analyse les captures, identifie les dépendances bloquantes, reconstruit des balises depuis un JSON d'archive, maintient le fil sur plusieurs heures.
