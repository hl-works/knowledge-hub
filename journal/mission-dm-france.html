Cobra · Architecte de mission pour Claude Code

Brief mission Claude Code : vérification factures D&M France

Claude · Cobra · mai 2026 · Cobra
agenten-cours

Contexte

14 factures PDF D&M France importées en brouillon Odoo via IA native. Les brouillons existent mais ne sont pas fiables : quantités, prix, DEEE et escompte peuvent être faux ou absents, le lien aux PO n'est pas garanti.

Ce qui a été construit

  • Brief MISSION_DM_FRANCE_FACTURES.md prêt à donner à Claude Code
  • Requêtes XML-RPC documentées : brouillons, lignes, correction, recherche PO
  • Logique de matching PDF ↔ brouillon via référence fournisseur
  • Stratégie DEEE (chercher produit existant) et escompte (chercher compte sur factures D&M validées)
  • Règles de sécurité : jamais valider, tout logger, signaler les doutes plutôt que deviner

Ce qui était difficile

  • Le matching PO est le point le plus fragile : l'IA Odoo ne le fait pas fiablement — fallback nécessaire (numéro PO → date/montant → références produits)
  • DEEE et escompte souvent absents des brouillons IA (format variable)

Stack

Python, XML-RPC Odoo, Claude Code, Odoo 18 (account.move, account.move.line, purchase.order, product.product).

Ce que ça illustre

Claude comme architecte de mission pour Claude Code : pas d'exécution directe faute de MCP Odoo, mais production d'un brief structuré, complet et sécurisé qui permet à Claude Code de travailler en autonomie sur une tâche comptable sensible avec des garde-fous clairs.
