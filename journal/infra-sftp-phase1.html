Cobra · Infra + stratégie produit

Infrastructure SFTP Cobra : phase 1 setup serveur

Claude · Cobra · 21 mai 2026 · Cobra
infraen-cours

Contexte

Cobra reçoit les fichiers de stock fournisseurs (Sogetronic, Dimetel, AV Industrie…) manuellement ou par email. GitHub Actions testé pour Dimetel puis abandonné. Objectif : infra SFTP pérenne où chaque fournisseur dépose ses fichiers, récupérés automatiquement par Odoo via cobra_import_stock_drop.

Ce qui a été construit

  • Décision stack : Scaleway PLAY2-PICO PAR1, Ubuntu 26.04, ~0,019€/heure
  • Instance cobra-ftp opérationnelle (IP 51.15.223.154), clé SSH enregistrée, connexion root OK
  • apt update && upgrade, kernel à jour
  • Plan de sécurité validé : fail2ban, ufw, SFTP chroot, structure /srv/sftp/

Ce qui était difficile

  • Clé SSH ajoutée après création d'instance non injectée — reboot nécessaire
  • La console Scaleway in-browser est une CLI Scaleway, pas un terminal serveur — confusion initiale

Stack

Ubuntu 26.04, OpenSSH SFTP, fail2ban, ufw, Scaleway PLAY2-PICO, Python/XML-RPC (cron à venir), cobra_import_stock_drop Odoo 18.

Ce que ça illustre

Session infra + stratégie produit : choix d'hébergement, arbitrage technique (FTP vs email vs portail), setup guidé sans compétence sysadmin requise. Claude comme copilote opérationnel au-delà du seul code.
