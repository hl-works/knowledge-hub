C'est quoi une « loop » ? Les tâches récurrentes d'agent

Un agent classique travaille vers un objectif puis s'arrête. Une loop, c'est l'étage au-dessus : la même tâche relancée à intervalle régulier — surveiller, vérifier, alerter — sans toi. Je suis en train d'en déployer chez Cobra ; voilà le concept, et le cadre.

En 30 secondes

  • Une loop = un prompt relancé à intervalle régulier par un agent (toutes les 5 min, toutes les heures…), pour surveiller ou vérifier quelque chose sans toi.
  • Dans Claude Code, la commande /loop fait ça en une ligne — avec un garde-fou intégré : expiration automatique au bout de 3 jours.
  • Règle simple : besoin ponctuel (< 3 jours) → loop ; besoin permanent → vraie tâche planifiée (cron, GitHub Actions).

1. La loop, l'étage au-dessus de l'agent

Dans C'est quoi un agent IA, j'expliquais la différence assistant/agent : l'assistant répond, l'agent agit en boucle jusqu'au résultat — puis s'arrête. La loop ajoute la dimension temps : la tâche entière est relancée à intervalle régulier. L'agent ne travaille plus une fois vers un objectif, il revient vérifier — encore et encore.

C'est le pattern naturel de tout ce qui est surveillance : on ne « termine » jamais de surveiller un déploiement ou un flux de commandes ; on regarde, on compare à l'état attendu, on alerte si ça dévie, et on revient plus tard.

2. Comment ça marche dans Claude Code

Claude Code (l'outil que j'utilise pour tout le dev — voir Chat, Cowork ou Code) embarque une commande /loop : tu donnes un prompt et un intervalle (en secondes, minutes, heures ou jours), et il relance ce prompt à chaque échéance, dans ta session.

Le détail qui m'a fait adopter le truc : le garde-fou est intégré. Une loop expire automatiquement 3 jours après sa création — elle tourne une dernière fois, puis se supprime. Une loop oubliée ne peut pas tourner indéfiniment ni cramer un budget en silence. C'est exactement le genre de cadre que je réclamais dans le guide sur les agents (objectif, budget, fréquence, critère de succès) — sauf qu'ici, la borne est posée par l'outil lui-même.

3. Loop ou tâche planifiée ?

Les deux relancent une tâche à intervalle régulier ; tout le reste diffère.

Ma veille IA quotidienne, par exemple, est une tâche planifiée (elle doit tourner tous les matins, pour toujours). Surveiller qu'un déploiement Odoo se passe bien cet après-midi, c'est une loop : utile trois heures, oubliable ensuite.

4. À quoi ça sert chez un marchand

Tout ce qui se vérifie à intervalle régulier sur une journée ou deux :

  • Commandes : vérifier toutes les 30 minutes que les commandes du matin partent bien en préparation, alerter si un statut bloque.
  • Mise en production : après un déploiement, contrôler toutes les 10 minutes que les pages clés répondent et que rien ne casse.
  • Flux fournisseurs : surveiller qu'un import de stock attendu est bien arrivé et s'est bien intégré.
  • Réappro critique : suivre une référence en rupture et prévenir dès que le stock remonte.

C'est ce chantier que j'ouvre en ce moment chez Cobra : faire passer des vérifications que je faisais à la main (ou pas du tout) dans des loops. L'entrée de journal détaillée arrivera quand les premières tourneront en prod — comme toujours, tout sera documenté.
